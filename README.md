# 🗂️ Dossier

### Autonomous agentic job search — built for engineers, not recruiters.

[![Python](https://img.shields.io/badge/python-3.12+-blue.svg)](https://www.python.org/downloads/)

[![Claude](https://img.shields.io/badge/LLM-Claude%20Sonnet-orange)](https://anthropic.com/)

Dossier finds, scores, and ranks the most relevant ML/AI jobs for you — automatically.
It reads your profile once, searches across multiple job boards in parallel, eliminates noise
with rule-based pre-filters, and sends only the best opportunities through a paid LLM scorer.
You get a ranked terminal table with direct links and urgency tiers. Apply to the right jobs,
not all of them.

## ✨ Features

- **Profile-driven, not hardcoded** — swap `profile/profile.json` for any user with zero code changes
- **Two-pass scoring** — rule-based pre-filter (free) eliminates ~60% of jobs; LLM scores only what survives
- **Parallel scoring** — 8-worker `ThreadPoolExecutor` takes serial 10-min scoring down to ~2 min
- **Verified company tiers** — 50 watchlist companies (MAANG, top Indian product, top AI startups) scored from ground truth, not LLM inference
- **Seniority-aware** — experience band computed at your planned switch date; penalises over/under-levelled roles automatically
- **Urgency tiers** — URGENT / HIGH / NORMAL / LOW from posting date (24h applications get 3× response rate)
- **Full audit trail** — every rejected job saved to `data/rejected_jobs.json` with score, reason, and description preview
- **Multi-source** — Indeed + Glassdoor (via JobSpy) + LinkedIn guest API, 10 search terms each

## How It Works

```
~550 raw jobs (Indeed + LinkedIn × 10 search terms)
        │
        ▼  is_hard_no()              service companies, aggregators → skip
        ▼  description < 100 chars   no content → skip
        ▼  is_seniority_mismatch()   profile-driven: penalises Senior/Staff/VP etc.
        ▼  classify_job_function()   support_ops → skip before LLM
        │
        ├── ~60% eliminated with zero LLM spend
        │
        ▼  ThreadPoolExecutor (8 workers)
           score_job() × remaining jobs
           ├─ company_tier  ← stated fact from target_companies.json (no guessing)
           ├─ job_function  ← stated fact from title keywords (no guessing)
           └─ GPT-5.4-mini → score 1–10 · reason · skill gaps
        │
        ▼  min_score gate · parent-company diversity cap (max 5 per company)
        │
        └── Ranked terminal table with urgency, score, link
```

**Cost per full run:** ~$0.03–0.06 (GPT-5.4-mini at $0.75/M input, ~100 jobs scored)

## Quick Start

**Requirements:** Python 3.12+, [uv](https://docs.astral.sh/uv/), OpenAI API key, Anthropic API key

```bash
# 1. Clone
git clone https://github.com/<your-username>/dossier.git && cd dossier

# 2. Install
uv sync

# 3. Configure
cp .env.example .env   # then fill in OPENAI_API_KEY and ANTHROPIC_API_KEY

# 4. Add your profile
#    See profile/profile.json — fill in identity, target roles, skills

# 5. Run
python scripts/run_job_discovery.py --hours 240
```

## Usage

```bash
# Discover jobs from the last 10 days (default min score: 5/10)
python scripts/run_job_discovery.py --hours 240

# Stricter filter — only high-relevancy jobs from last 3 days
python scripts/run_job_discovery.py --hours 72 --min-score 7

# Build or rebuild your profile from resume + interview
python scripts/run_persona_builder.py

# Check LLM connectivity and model health
python tests/test_llm_client.py
```

**Output files after each run:**
| File | Contents |
|---|---|
| `data/last_discovery_run.json` | All scored jobs above threshold, full detail |
| `data/rejected_jobs.json` | Every filtered job with reason and description preview |
| `data/artifacts/{job_id}/` | Per-job JD text + scorecard JSON |

## Configuration

All config lives in two files:

**`.env`** — API keys (copy from `.env.example`)
```env
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
```

**`profile/profile.json`** — your persona (generated by `run_persona_builder.py` or written manually)
```json
{
  "identity": { "name": "...", "total_experience_months": 20, ... },
  "target": {
    "roles": ["MLE-1", "AI Engineer", "Data Scientist"],
    "locations": ["Bengaluru"],
    "min_salary_lpa": 25,
    "switch_timeline_months": 8,
    "search_terms": ["Machine Learning Engineer", "LLM Engineer", ...]
  },
  "skills": [{ "skill": "PyTorch", "depth": "can_architect", "market_aliases": ["torch"] }]
}
```

**`profile/target_companies.json`** — 50 watchlist companies with verified tiers, LinkedIn slugs, and ATS info

## LLM Strategy

| Task | Model | Why |
|---|---|---|
| Job scoring (per job) | `gpt-5.4-mini` | Runs on every job — cost is the constraint |
| Company intel, referral ranking | `gpt-5.4-mini` | Noisy input needs reasoning, not premium writing |
| Persona interview, cold messages | `gpt-5` | Conversational quality matters |
| Cover letter generation | `claude-haiku-4-5` | Good writing at efficient cost |
| Resume bullet rewriting (LaTeX) | `claude-sonnet-4-6` | Best code + writing precision, LaTeX-aware |

All model names are constants in `config.py`. Changing any model is a one-line edit.
