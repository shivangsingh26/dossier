# M8 — Resume Tailor + Cover Letter (Scoped Stub)

> **Status:** Stub. Say "expand M8".

**Spec:** [docs/superpowers/specs/2026-05-18-dossier-saas-frontend-design.md](../specs/2026-05-18-dossier-saas-frontend-design.md) §7 (Max tier), §10 (`/resume`), agent already exists in `dossier_sdk.agents.resume_agent`.

**Goal:** Max tier feature. "Tailor for this job" button on JobCard → backend runs 3-pass resume agent → LaTeX compiled → PDF returned. Frontend preview pane + download. Cover letter on separate button.

## Files (frontend)

- `frontend/app/(app)/resume/page.tsx` (library of generated resumes per job)
- `frontend/components/dossier/jobs/TailorResumeButton.tsx` (in JobDetail)
- `frontend/components/dossier/resume/ResumePreview.tsx` (PDF embed)
- `frontend/components/dossier/resume/CoverLetterEditor.tsx`

## Files (backend)

- `backend/src/dossier_api/routers/resume.py`
  - `POST /jobs/{id}/tailor-resume` (12 credits, Max only)
  - `POST /jobs/{id}/cover-letter` (4 credits)
  - `GET /jobs/{id}/resume.pdf`
  - `GET /resume/library`
- Wire `dossier_sdk.agents.resume_agent` into worker (includes Pass 1/2/3 + pdflatex compile)
- Server must have `pdflatex` installed — document in backend README

## Acceptance criteria

- Max user clicks "📄 Tailor resume" on a job → 12 credits → progress → PDF appears in 60–120s
- Preview pane shows PDF, "Download" button works
- Cover letter button → 4 credits → text editor with generated draft
- Resume library shows all generated PDFs grouped by job
- pdflatex failure → credits refunded, error shown
- `pytest backend/tests/test_resume.py -v` green
