# M9 — Referral Finder + Cold Message (Scoped Stub)

> **Status:** Stub. Say "expand M9".

**Spec:** [docs/superpowers/specs/2026-05-18-dossier-saas-frontend-design.md](../specs/2026-05-18-dossier-saas-frontend-design.md) §7 (Max tier), §10 (`/referrals`), agent in `dossier_sdk.agents.referral_finder`.

**Goal:** Max tier. "Find referrals" on JobCard → backend searches LinkedIn + Tavily for relevant contacts at target company → ranked list returned. Per contact, generate cold message draft. User sends manually (we never send).

## Files (frontend)

- `frontend/app/(app)/referrals/page.tsx` (contacts grouped by job)
- `frontend/components/dossier/jobs/FindReferralsButton.tsx`
- `frontend/components/dossier/referrals/ContactCard.tsx` (name, role, linkedin, mutual signal, message draft)
- `frontend/components/dossier/referrals/MessageEditor.tsx` (edit + copy + "mark sent")

## Files (backend)

- `backend/src/dossier_api/routers/referrals.py`
  - `POST /jobs/{id}/find-referrals` (6 credits)
  - `POST /referrals/{contact_id}/draft-message` (2 credits)
  - `POST /referrals/{contact_id}/mark-sent`
  - `GET /referrals?job_id=`

## Acceptance criteria

- Max user clicks "🤝 Find referrals" → 6 credits → 5–10 ranked contacts appear
- Each contact has: name, role, LinkedIn URL, why-good signal (mutual connection / alumni / team match)
- "Draft message" → 2 credits → personalized cold message in user's tone
- "Mark sent" persists to backend so we can later track responses (M10+)
- Tier gate visible for Pro and Lite users
- `pytest backend/tests/test_referrals.py -v` green
