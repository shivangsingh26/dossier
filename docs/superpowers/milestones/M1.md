# M1 — Frontend Skeleton + Clerk Auth + Theme Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Spec:** [docs/superpowers/specs/2026-05-18-dossier-saas-frontend-design.md](../specs/2026-05-18-dossier-saas-frontend-design.md) §3 (brand), §5 (stack), §6 (auth), §10 (routes/components), §15 M1 (acceptance).

**Goal:** Stand up the Next.js 16 frontend with locked warm-mocha theme, Split-D brand identity, Clerk auth, marketing surface (hero + pricing + footer), themed sign-in/sign-up, and an empty post-login app shell with sidebar + CreditPill placeholder.

**Architecture:** Next.js 16 App Router project under `frontend/`, TypeScript strict, Tailwind v4 (CSS-only `@theme` directive — no tailwind.config.ts), shadcn/ui new-york style, motion, `@clerk/nextjs` for hosted auth UI, TanStack Query + RHF + zod installed for M3/M4. Brand mark + wordmark are pure SVG React components. Marketing pages are RSC + statically rendered. App pages live under `(app)` route group behind Clerk middleware. Theme tokens (warm mocha) declared in `app/globals.css` via Tailwind v4 `@theme`. Fonts via `next/font/google` (Fraunces, Geist, Geist Mono).

**Tech Stack:** pnpm, Node.js ≥ 22 LTS, Next.js 16, TypeScript strict, Tailwind v4, shadcn/ui (new-york), motion, `@clerk/nextjs`, `@tanstack/react-query`, `react-hook-form`, `zod`, Vitest + `@testing-library/react`.

**Estimated time:** 6–10 hours (1–2 sessions, 12 tasks).

## Notes for the executor

1. **TDD adapted for frontend:** unit tests cover only stable surface (brand SVG fills, theme tokens). Visual UI verified manually + via `pnpm build` smoke. Don't over-test markup.
2. **Tailwind v4 quirk:** `@theme` block lives in `globals.css`, NOT in a separate `tailwind.config.ts`. Class names like `bg-[color:var(--color-bg)]` reference CSS vars directly — this is idiomatic v4 + works with shadcn.
3. **Clerk free tier:** 10K MAU is plenty for closed beta. If you exceed, upgrade or migrate to Auth.js. Free tier supports Google/GitHub OAuth out of the box — enable in Clerk dashboard.
4. **next/font automatically inlines fonts** — no manual `<link>` tags. Fraunces axes (opsz, SOFT, WONK) enable optical sizing + variable italic, which is what makes "Dossier." look properly typeset.
5. **Don't add ai-elements yet** — M3's chat quiz will. Adding it now bloats install.
6. **Mobile-first means write base classes for 375px, then layer `sm:` / `md:` / `lg:`.** The hero uses `clamp(2.25rem, 6vw, 4.5rem)` for fluid headline scaling — no breakpoints needed.
7. **If a step fails:** stop, read the error, fix the cause. Most common failure = Clerk middleware matcher excluding a route you needed protected. Check `middleware.ts` config block.
