# M1 — Frontend Skeleton + Clerk Auth + Theme (Scoped Stub)

> **Status:** Stub — flesh out into bite-sized tasks when M0 is complete. Open this file in a new session and say "expand M1".

**Spec:** [docs/superpowers/specs/2026-05-18-dossier-saas-frontend-design.md](../specs/2026-05-18-dossier-saas-frontend-design.md) §3 (brand), §5 (stack), §6 (auth), §10 (routes/components).

**Goal:** Stand up the Next.js 16 frontend with locked theme, Clerk auth, marketing site (hero + pricing + sign-in/sign-up), and an empty post-login app shell. No backend integration yet.

**Architecture:** `frontend/` becomes a Next.js 16 App Router project with TS strict mode, Tailwind v4, shadcn/ui, motion, Clerk, TanStack Query, react-hook-form + zod. Theme tokens (warm mocha) live in `globals.css` + `tailwind.config.ts`. Brand mark + wordmark are SVG components. Marketing pages SSG. App pages CSR with Clerk guard.

**Tech additions:** pnpm, Node.js 22 LTS, Next.js 16, Tailwind v4, shadcn-ui CLI, `@clerk/nextjs`, `@tanstack/react-query`, `motion`, `react-hook-form`, `zod`.

**Estimated time:** 6–10 hours (1–2 sessions).

## Acceptance criteria (from spec §15 M1)

- Visit `localhost:3000` → marketing hero with locked warm-mocha theme + Split-D mark
- `/pricing` shows 3 tiers (Lite / Pro / Max) with credit counts
- Click "Sign up" → Clerk hosted UI (themed mocha) → complete signup
- Land on `/dashboard` showing sidebar + brand lockup + "Welcome, {firstName}" heading + CreditPill (static "100 / 100" placeholder)
- Theme tokens match spec §3.1 exactly
- Page works at 1440, 768, 375 widths (manually verify in DevTools)
- No TypeScript errors (`pnpm typecheck` passes)
- No console errors in browser (`pnpm dev` clean)
- Lighthouse mobile score ≥ 85 for `/`
