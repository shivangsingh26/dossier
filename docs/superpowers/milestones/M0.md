# M0 — Repo Restructure + SDK Packaging Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Spec:** [docs/superpowers/specs/2026-05-18-dossier-saas-frontend-design.md](../specs/2026-05-18-dossier-saas-frontend-design.md) §4, §11 (M0), §15 (M0 acceptance).

**Goal:** Restructure repo into `sdk/` + `backend/` + `frontend/` folders without breaking the existing pipeline. Package existing Python code as `dossier-sdk`, an installable, importable library.

**Architecture:** Move every Python module under `/agents`, `/core`, `/prompts`, plus `config.py`, into `sdk/src/dossier_sdk/`. Create a `sdk/pyproject.toml` declaring `dossier-sdk` v1.0.0. Update `dashboard.py`, `run_dossier.py`, and every `scripts/run_*.py` to import from `dossier_sdk.*`. Create empty `backend/` + `frontend/` skeletons (with .gitkeep) so subsequent milestones drop into ready slots. Existing pipeline must keep running end-to-end at every step — verified by smoke test.

**Tech Stack:** uv, pytest, Python 3.12, ruff. No new runtime dependencies.

**Estimated time:** 2–4 hours.

```python
"""
run_dossier.py — CLI entry point for the Dossier pipeline.

This file is a thin argparse wrapper around dossier_sdk.orchestrator.
All pipeline logic lives in the SDK.
"""
from __future__ import annotations
import argparse
import sys
from dossier_sdk.config import Config
from dossier_sdk.orchestrator import (
    run_pipeline,
    run_discovery,
    run_watchlist,
    run_company_intel,
    run_market_intel,
    run_gap_analysis,
)


def main(argv: list[str] | None = None) -> int:
    parser = argparse.ArgumentParser(description="Run the Dossier pipeline.")
    parser.add_argument("--user", default="shivang", help="User slug (default: shivang)")
    parser.add_argument("--mode", default="full",
                        choices=["quick", "full", "company-intel", "market-intel", "watchlist"])
    parser.add_argument("--hours", type=int, default=24)
    parser.add_argument("--min-score", type=int, default=5)
    parser.add_argument("--with-referrals", action="store_true")
    args = parser.parse_args(argv)

    config = Config(user=args.user)

    if args.mode == "company-intel":
        results = [run_company_intel(config, min_score=args.min_score)]
    elif args.mode == "market-intel":
        results = [run_market_intel(config)]
    elif args.mode == "watchlist":
        results = [run_watchlist(config, hours=args.hours, min_score=args.min_score)]
    else:
        results = run_pipeline(
            config,
            mode=args.mode,
            hours=args.hours,
            min_score=args.min_score,
            with_referrals=args.with_referrals,
        )

    failed = [r for r in results if not r.success]
    return 1 if failed else 0


if __name__ == "__main__":
    sys.exit(main())
```

**Important:** If the existing `run_dossier.py` does anything special the orchestrator does not yet handle (e.g. writing a `last_*_run.json` file, sending Telegram alerts), copy that logic into the orchestrator functions, not into the CLI wrapper.

## Notes for the executor

1. **Don't skip the smoke tests after each task.** A broken import in core.py won't surface until an agent tries to call it. The end-to-end run at each task gate catches this immediately.
2. **`git mv` not `mv`.** This preserves file history and makes the diff readable.
3. **`sed -i ''` on macOS, `sed -i` on Linux.** The bash blocks assume macOS (the user's environment).
4. **If a step fails:** stop, do not push forward, read the error fully, fix the cause. Most common cause = sed didn't match a `from X` line because the import used `import X` style instead — handle by hand.
5. **No new features in M0.** Anything that says "while we're at it" — say no. Move it to M1+ or a separate cleanup PR.
