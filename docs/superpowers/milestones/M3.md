# M3 — Persona Builder Wizard (Scoped Stub)

**Spec:** [docs/superpowers/specs/2026-05-18-dossier-saas-frontend-design.md](../specs/2026-05-18-dossier-saas-frontend-design.md) §10 (routes), §9 (`/persona/*` endpoints), §3 (brand for the wizard).

**Goal:** Replace the terminal `persona_builder.py` interview + `export_questionnaire.py` form with a 4-step web wizard. Backend wraps the existing `dossier_sdk.agents.persona_builder` synthesis.

**Architecture:** Frontend wizard component with URL-hash-tracked steps. State held client-side; each step POSTs to FastAPI which persists incrementally. Final synthesis (LLM call) runs as a background task on the worker (so it doesn't block the request). Frontend polls/SSE for completion.

## 4-step flow

1. **Upload** — Resume PDF + LinkedIn PDF + writing samples (optional). Saved to `data/{slug}/raw/`.
2. **Targets** — Form: target roles, locations, salary floor, switch timeline, remote OK, hard nos. Saved to `profile/{slug}/questionnaire.json`.
3. **Quiz** — Chat-style 12-question interview using AI Elements components. Each user message POSTs `/persona/quiz-message`, agent's next question streamed back via SSE. Answers persisted to `profile/{slug}/quiz_answers.json`.
4. **Review** — Show synthesized persona JSON. User can edit any field inline. Click "Finalize" → `POST /persona/finalize` → worker runs `persona_builder.synthesize()` → final `profile.json` written.

## Acceptance criteria (from spec §15 M3)

- Fresh signup → redirected to `/onboarding` (because no `profile.json` for that user yet)
- User uploads 2 PDFs → "Step 1 done" → moves to step 2
- User fills questionnaire form → step 3 starts
- User answers 12 chat questions, each response streams in
- Step 4 shows synthesized persona; user edits a field, clicks "Finalize"
- 30–60s later, `profile/{slug}/profile.json` exists and matches schema
- User redirected to `/dashboard`
- `pytest backend/tests/test_persona.py -v` green
- Persona synthesis does NOT debit credits (it's part of onboarding, free for everyone)
