**Goal:** Admin can approve pending signups, change tiers/credits, trigger pipelines for any user, edit `target_companies.json`, view waitlist.

- `frontend/app/(admin)/admin/page.tsx` (role-gated layout)

- `frontend/app/(admin)/admin/waitlist/page.tsx`
- `frontend/app/(admin)/admin/companies/page.tsx` (JSON editor)
- `frontend/components/dossier/admin/UserActions.tsx` (approve, suspend, tier change, credit topup)
- `frontend/components/dossier/admin/JsonEditor.tsx`

## Files to be created (backend)

- `backend/src/dossier_api/routers/admin.py` — all `/admin/*` endpoints (see spec §9)
- `backend/src/dossier_api/services/admin_service.py`
- File lock helper for concurrent `target_companies.json` edits (fasteners)

## Acceptance criteria (from spec §15 M5)

- Admin (shivang) logs in → sidebar shows "Admin" link
- `/admin` lists all users with status, tier, credits, last run
- Pending user → "Approve" → flips to active → email notification
- Tier dropdown changes tier; credits input changes credits (with reason logged in `credit_log`)
- Companies editor saves JSON with schema validation
- Non-admin user gets 403 on `/admin` and on `/admin/*` API calls
- `pytest backend/tests/test_admin.py -v` green
